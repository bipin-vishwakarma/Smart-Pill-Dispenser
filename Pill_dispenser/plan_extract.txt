Project Overview The Automatic Pill Dispenser is a portable, battery-powered medical device that dispenses medication on a programmed schedule. It uses a servo motor to rotate pill compartments, alerting the patient via buzzer and LED indicators when it is time to take medication. 1.1 What the device does Stores pills in a rotating compartment wheel (servo-driven) Keeps track of medication schedule via ESP32 RTC or BLE phone sync At scheduled time: rotates servo to correct compartment, buzzes, lights LED User presses button to confirm pill taken — LED turns off Runs on rechargeable Li-Ion battery, charges via USB-C OLED display shows current time, next dose time, and battery level 1.2 Miniaturization goals — V1 to V3 Feature V1 (Tinkercad) V2 (KiCad) V3 (Target) Microcontroller Arduino Uno ATmega328P ESP32-S3 (wrong footprint) ESP32-S3-WROOM-1 (correct) Form factor Breadboard ~20×10cm PCB ~70×55mm (no outline) PCB 60×50mm (clean) Power USB-A wall only USB-C + Li-Ion USB-C + Li-Ion (fixed) Display 16×2 LCD, 8 wires I2C OLED, 2 wires I2C OLED, 2 wires Buzzer THT 12mm THT 12mm (V2 mistake) SMD 5020 (5×5mm) Servo power 5V correct 3.3V (WRONG - bug) VBUS 5V (fixed) LED resistors 470Ω Inconsistent: 100Ω–1kΩ All 330Ω (standardized) Wireless None BLE + WiFi (ESP32-S3) BLE + WiFi (ESP32-S3) Fab ready? V2 Engineering Review — What We&apos;re Fixing 2.1 Critical bugs (board won&apos;t function without fixing these) Bug 1: Servo connected to 3.3V rail — WRONG The servo motor in V2 is powered from the 3.3V LDO output. This is incorrect and will cause the board to malfunction in the field. Servo motors draw 500–900mA at stall and 150–300mA during movement The AP2112K LDO is rated for 600mA total — shared with the ESP32 (~200mA WiFi peak) When the servo moves, it will brown out the 3.3V rail, crashing the ESP32 FIX: Connect servo VCC directly to VBUS (5V from USB-C). Add 100µF decoupling cap on the 5V servo line. Only the PWM signal wire goes to ESP32 GPIO. Bug 2: ESP32 symbol/footprint mismatch The schematic uses ESP32-S3-MINI-1 symbol but the PCB has ESP32-S3-WROOM-1 footprint. These are physically different modules. MINI-1: 27mm × 34.3mm, 61 pads WROOM-1: 18mm × 20mm, 38 pads — smaller and more common FIX: Change schematic symbol to RF_Module:ESP32-S3-WROOM-1 to match PCB footprint Bug 3: LED current limiting resistors are inconsistent V2 has three different resistor values for LEDs: R1=100Ω, R2=470Ω, R13/R14=1kΩ. This is a design error. At 3.3V with 100Ω: (3.3 - 2.0) / 100 = 13mA — borderline; blue/white LEDs will be over-driven At 3.3V with 1kΩ: (3.3 - 2.0) / 1000 = 1.3mA — too dim, barely visible FIX: Standardize ALL LED current limiting resistors to 330Ω. This gives ~4mA which is clean and bright for status indication. 2.2 Important improvements (reliability and size) Issue 4: AMS1117-3.3 is redundant AP2112K-3.3 already provides 3.3V at 600mA — sufficient for ESP32 + peripherals Having two LDOs on the same output rail risks reverse current and unstable regulation FIX: Remove AMS1117 from V3. If more current is needed later, upgrade AP2112K to AP2112K (1A variant) Issue 5: 470µF THT electrolytic capacitor (C10) A large radial electrolytic sitting on an SMD board defeats the miniaturization goal It adds significant board height and is a mechanical weak point FIX: Replace with 100µF SMD electrolytic in 6.3×5.4mm package, or 2× 47µF 0805 ceramic Issue 6: 12mm THT buzzer (BZ1) The original Buzzer_12x9.5RM7.6 footprint is massive for an SMD board FIX: Replace with MLT-5020 or similar SMD magnetic buzzer, 5×5mm footprint, same driving circuit (MMBT2222A stays) Issue 7: Insufficient decoupling on 3.3V near ESP32 ESP32-S3 WiFi/BLE radio causes brief ~300mA transient current spikes on 3.3V Only 4× 100nF bypass caps present — no bulk capacitance FIX: Add 1× 10µF ceramic 0805 cap on 3.3V rail, placed within 5mm of ESP32 module VDD pins 2.3 What&apos;s already correct in V2 — keep these Component Status Notes TP4056 charger CORRECT Standard LiPo/Li-Ion charger IC, proven design AP2112K-3.3 LDO CORRECT 600mA, SOT-23-5, low dropout — good choice 1N5819 Schottky diode CORRECT Reverse polarity protection + power OR-ing CC1/CC2 resistors 5.1kΩ CORRECT Required for USB-C 5V/500mA negotiation per spec I2C pull-ups R6/R7 4.7kΩ CORRECT Correct for 3.3V I2C at standard 100/400kHz MMBT2222A buzzer driver CORRECT GPIO can&apos;t sink buzzer current directly — transistor needed BOOT + EN pull-up 10kΩ CORRECT Matches Espressif reference design exactly 100nF bypass caps ×4 CORRECT Keep, but add 10µF bulk as mentioned above Antenna keep-out zone CORRECT No copper under ESP32 antenna — good RF practice JST-PH battery connector CORRECT Standard, locking, correct 2mm pitch R9/R10 100kΩ voltage divider CORRECT Battery level monitoring via ADC — document net labels 3. Critical routing rules USB D+/D− differential pair Route GPIO19 (D−) and GPIO20 (D+) as a differential pair Keep traces equal length — maximum 5mm length mismatch Minimum 0.2mm spacing between the two traces No vias on D+/D− — route entirely on front copper Keep away from high-current power traces ESP32 antenna keep-out The WROOM-1 module has a PCB antenna at the top edge No copper (traces, pours, fills) on either layer under the antenna region Keep-out extends 3mm beyond the module edge where the antenna protrudes This is already correctly set in V2 — preserve in V3 Decoupling capacitors Place each 100nF bypass cap within 2mm of its corresponding VDD pin on the ESP32 footprint Route cap GND directly to ground pour with a short via — not through a long trace 10µF bulk cap can be up to 5mm away from the module Power traces 3.3V rail: minimum 0.5mm trace width VBUS 5V rail: minimum 0.8mm trace width (servo can pull 500mA) Battery input: minimum 0.5mm trace width GND: use copper pour — do not route individual GND traces Servo connector Place J2 (servo) on a board edge for easy cable routing to the mechanical assembly The 5V trace from VBUS to servo VCC pin must be at least 0.8mm wide Firmware Notes 7.1 Programming via USB-C (same port) The ESP32-S3 has native USB. GPIO19 = D−, GPIO20 = D+ connect directly to the USB-C connector. No CH340, CP2102, or any USB-to-UART chip needed. 7.2 Flash procedure To enter bootloader mode and flash firmware: Hold BOOT button (SW2 — GPIO0 to GND) Press and release RESET button (SW3 — EN to GND) Release BOOT button — ESP32 is now in download mode Select correct COM port in Arduino IDE or run esptool After flash completes, press RESET once to run new firmware 7.3 Arduino IDE setup for ESP32-S3 Install ESP32 board package: https://raw.githubusercontent.com/espressif/arduino-esp32/gh-pages/package_esp32_index.json Board: ESP32S3 Dev Module USB Mode: USB-OTG (TinyUSB) Upload Mode: UART0 / Hardware CDC Flash Size: 8MB (WROOM-1 default) 7.4 Key GPIO assignments for firmware GPIO Function Notes GPIO0 BOOT button Active LOW — 10kΩ pull-up to 3.3V GPIO4 I2C SDA (OLED) 4.7kΩ pull-up on PCB GPIO5 I2C SCL (OLED) 4.7kΩ pull-up on PCB GPIO9 Servo PWM 50Hz, 1–2ms pulse width GPIO10 Buzzer (via Q1) Drive HIGH to activate buzzer GPIO11 STATUS LED Drive HIGH = LED ON (330Ω in series) GPIO12 USER button Active LOW — add pull-up in firmware GPIO19 USB D− Do NOT use as GPIO — reserved for USB GPIO20 USB D+ Do NOT use as GPIO — reserved for USB GPIO34/35 Battery ADC 100kΩ voltage divider — read battery level EN RESET Active LOW — 10kΩ pull-up to 3.3V 8.
